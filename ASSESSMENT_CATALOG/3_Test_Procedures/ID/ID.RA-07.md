# ID.RA-07: Change and Exception Risk Management Test Procedures

**CSF Subcategory:** ID.RA-07 - Changes and exceptions are managed, assessed for risk impact, recorded, and tracked

---

## Test Procedures

1. **Review Change Management Process**
   - Obtain ServiceNow change management process documentation
   - Verify risk assessment is a required field in change requests
   - Confirm CAB review process for high-risk changes
   - Check change approval workflow and authorization levels

2. **Validate Change Risk Assessment**
   - Sample 5 recent change requests
   - Verify each includes completed risk assessment
   - Confirm high-risk changes received CAB review
   - Check post-implementation reviews were conducted

3. **Assess Change Compliance**
   - Review change management compliance metrics (85% target)
   - Investigate the 15% non-compliant changes
   - Check if emergency change process is used appropriately
   - Verify unauthorized changes are detected and remediated

4. **Review Exception Management**
   - Obtain exception request process documentation
   - Sample exception requests and verify risk documentation
   - Confirm management approval for each exception
   - Check that exception risks are periodically re-evaluated

5. **Evaluate Change Tracking and Reporting**
   - Verify all changes are recorded in ServiceNow
   - Check for change metrics reporting and trend analysis
   - Assess change failure rate tracking
   - Confirm change audit trail completeness

## Evidence Requests

- [ ] Change management process documentation
- [ ] ServiceNow change compliance report
- [ ] Sample change requests with risk assessments (5)
- [ ] CAB meeting minutes (recent 3)
- [ ] Exception request examples with approvals
- [ ] Change management metrics dashboard

## Notes

The 85% change compliance rate is the key finding. Investigate whether the 15% gap represents emergency changes, process circumvention, or specific teams/systems that bypass change management.
