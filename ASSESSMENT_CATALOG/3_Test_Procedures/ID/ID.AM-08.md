# ID.AM-08: Asset Lifecycle Management Test Procedures

**CSF Subcategory:** ID.AM-08 - Systems, hardware, software, services, and data are managed throughout their life cycles

---

## Test Procedures

1. **Review Lifecycle Management Policies**
   - Obtain SDLC documentation including security review gates
   - Review hardware provisioning and decommissioning procedures
   - Verify data retention and destruction policies
   - Check for end-of-life tracking processes

2. **Validate Change Management Integration**
   - Review ServiceNow change management compliance metrics
   - Sample 5 recent changes and verify lifecycle documentation
   - Check that security review is included in change workflow
   - Identify changes that bypassed formal process

3. **Assess Decommissioning Procedures**
   - Review hardware decommissioning procedure documentation
   - Verify data sanitization steps are included
   - Check for recent decommissioning examples and evidence
   - Confirm decommissioned assets are removed from inventory

4. **Review Shadow IT Detection**
   - Assess mechanisms for identifying unauthorized technology use
   - Review recent shadow IT discoveries and remediation
   - Check for automated detection capabilities
   - Verify policy for handling discovered shadow IT

5. **Validate Data Destruction**
   - Review data retention policy and destruction timelines
   - Check for data destruction verification records
   - Verify media sanitization procedures for hardware disposal
   - Confirm destruction methods align with data sensitivity

## Evidence Requests

- [ ] SDLC documentation with security gates
- [ ] Hardware decommissioning procedure
- [ ] Change management compliance report
- [ ] Data retention and destruction policy
- [ ] Data sanitization verification records
- [ ] End-of-life software tracking report

## Notes

The 85% change management compliance rate warrants investigation into the 15% gap. Determine whether bypassed changes follow emergency procedures or represent process failures.
