# ID.IM-01: Evaluation-Based Improvement Test Procedures

**CSF Subcategory:** ID.IM-01 - Improvements are identified from evaluations

---

## Test Procedures

1. **Review Self-Assessment Program**
   - Obtain self-assessment methodology documentation
   - Verify CSF assessment scope and coverage
   - Confirm assessment findings are documented with evidence
   - Check that current threats and TTPs are considered

2. **Validate Improvement Identification**
   - Review assessment findings and improvement recommendations
   - Verify improvements are prioritized and tracked
   - Confirm remediation plans are created for identified gaps
   - Check for improvement implementation evidence

3. **Assess Automated Compliance Monitoring**
   - Review AWS Config compliance monitoring configuration
   - Verify automated compliance rules cover key controls
   - Check compliance monitoring coverage gaps
   - Confirm non-compliance alerts are acted upon

4. **Review Third-Party Assessment History**
   - Check for independent audit or assessment reports
   - Verify plans for third-party assessment
   - Review peer assessments or benchmarking activities
   - Assess maturity of evaluation program

5. **Evaluate Assessment Cadence and Coverage**
   - Verify assessment frequency (annual minimum)
   - Check for event-driven assessment triggers
   - Confirm all critical services are in assessment scope
   - Review assessment program improvement over time

## Evidence Requests

- [ ] Self-assessment methodology documentation
- [ ] CSF assessment findings and recommendations
- [ ] AWS Config compliance monitoring reports
- [ ] Improvement tracking records
- [ ] Remediation plan status
- [ ] Third-party assessment plans or reports

## Notes

As the first structured CSF assessment, focus on methodology soundness and whether the organization has built a repeatable assessment process that can mature over time.
