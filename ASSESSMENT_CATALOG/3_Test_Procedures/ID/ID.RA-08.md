# ID.RA-08: Vulnerability Disclosure Management Test Procedures

**CSF Subcategory:** ID.RA-08 - Processes for receiving, analyzing, and responding to vulnerability disclosures are established

---

## Test Procedures

1. **Review Disclosure Intake Process**
   - Verify security contact email is published and monitored
   - Check for security.txt file publication (RFC 9116)
   - Review vulnerability disclosure program (VDP) documentation
   - Confirm disclosure intake channels are documented

2. **Validate Disclosure Handling**
   - Review the triage process for received disclosures
   - Verify response SLAs for acknowledging and processing disclosures
   - Sample recent disclosure reports and track handling
   - Confirm findings are validated and recorded

3. **Assess Supplier Vulnerability Sharing**
   - Review vulnerability information sharing with suppliers
   - Check for formal information sharing agreements
   - Verify processes for receiving supplier vulnerability notifications
   - Assess coordination procedures for shared vulnerabilities

4. **Review Disclosure Response**
   - Verify remediation timelines for disclosed vulnerabilities
   - Check for coordination with disclosers on fix timelines
   - Confirm disclosure resolution is communicated
   - Review processes for public disclosure decisions

5. **Evaluate Program Maturity**
   - Assess VDP scope and coverage
   - Check for bug bounty program or plans
   - Review disclosure handling metrics
   - Verify responsible disclosure practices

## Evidence Requests

- [ ] Security contact publication evidence
- [ ] Vulnerability disclosure program documentation
- [ ] Sample disclosure reports and handling records
- [ ] Supplier vulnerability sharing agreements
- [ ] Disclosure response timeline records
- [ ] security.txt file (if published)

## Notes

The absence of a formal VDP and security.txt represents a gap in industry best practices. Even without a bug bounty, publishing clear disclosure channels reduces risk of uncoordinated disclosure.
