# GV.RM-04: Risk Response Options Test Procedures

**CSF Subcategory:** GV.RM-04 - Strategic direction that describes appropriate risk response options is established and communicated

---

## Test Procedures

1. **Review risk response framework**
   - Request documentation of approved risk response options (accept, avoid, mitigate, transfer)
   - Verify criteria for selecting each response option are defined
   - Confirm response options are aligned with risk appetite and tolerance statements

2. **Validate response option implementation**
   - For each of the five documented risks, verify the chosen response option is documented
   - Check that R-01 (understaffing) uses both mitigation (hiring) and transfer (CrowdStrike MDR)
   - Verify cybersecurity insurance status and coverage decisions

3. **Examine shared responsibility documentation**
   - Review conditions under which shared responsibility models are acceptable
   - Verify cloud shared responsibility (AWS) is documented and understood
   - Check that outsourced security functions have clear responsibility delineation

4. **Interview CISO on risk response decision-making**
   - Ask how risk response options are selected for new risks
   - Verify decision authority levels for different risk magnitudes
   - Confirm communication of risk response decisions to affected stakeholders

## Evidence Requests

- [ ] Risk response framework or strategy document
- [ ] Risk register showing response options per risk
- [ ] Cybersecurity insurance policy or decision documentation
- [ ] Shared responsibility model documentation

## Notes

Alma demonstrates practical risk response diversification (R-01 uses both mitigation and transfer). Assessment should verify this approach is formalized as a framework rather than being ad hoc per risk.
