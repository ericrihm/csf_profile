# RC.RP-06_Ex1: After-Action Reporting

**Subcategory:** RC.RP-06 — The end of incident recovery is declared based on criteria, and incident-related documentation is completed

**NIST SP 800-53 Ref:** IR-04, IR-08

## Implementation Example

> Ex1: Prepare an after-action report that documents the incident itself, the response and recovery actions, and the lessons learned

## Alma Security Implementation

Alma produces post-incident reviews following security incidents, documenting the incident timeline, response actions, recovery steps, and lessons learned. The 2024 security incidents generated after-action documentation that drove trust-rebuilding initiatives and procedure updates. However, the post-incident review process is not formalized with a standard template, required sections, or completion timeline, leading to inconsistency across incidents.
