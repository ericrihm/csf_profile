# ID.AM-05: Asset Classification and Prioritization

**Subcategory:** ID.AM-05 — Assets are prioritized based on classification, criticality, resources, and impact on the mission
**NIST SP 800-53 Ref:** RA-02,RA-03,RA-09

## Implementation Example

> Ex1: Define criteria for prioritizing each class of assets

## Alma Security Implementation

Alma Security has defined crown jewel assets and maintains a tiered asset classification scheme (Critical, High, Medium, Low). Business impact analysis informs asset prioritization. AWS resources are tagged with criticality levels. Asset priorities are reviewed during quarterly risk reviews.
