# GV.PO-02 Ex4: Update Policy for Technology Changes

**Subcategory:** GV.PO-02 — Policy for managing cybersecurity risks is reviewed, updated, communicated, and enforced to reflect changes in requirements, threats, technology, and organizational mission
**NIST SP 800-53 Ref:** AC-01, AT-01, AU-01, CA-01, CM-01, CP-01, IA-01, IR-01, MA-01, MP-01, PE-01, PL-01, PM-01, PS-01, PT-01, RA-01, SA-01, SC-01, SI-01, SR-01

## Implementation Example

> Update policy to reflect changes in technology (e.g., adoption of artificial intelligence) and changes to the business (e.g., acquisitions).

## Alma Security Implementation

Alma Security's Architecture Review Board flags technology changes (new SaaS adoptions, AI/ML integrations, cloud migration milestones) that require policy updates. The CISO evaluates each technology change against existing policies and initiates revisions as needed. Recent example: adoption of AI-assisted code review triggered updates to the acceptable use and data classification policies.
