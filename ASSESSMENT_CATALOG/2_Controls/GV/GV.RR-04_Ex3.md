# GV.RR-04 Ex3: Conduct Background Checks for Sensitive Roles

**Subcategory:** GV.RR-04 — Cybersecurity is included in human resources practices
**NIST SP 800-53 Ref:** PM-13, PS-01, PS-07, PS-09

## Implementation Example

> Conduct background checks prior to onboarding new personnel for sensitive roles, and periodically for personnel in those roles.

## Alma Security Implementation

All personnel with access to customer data, production systems, or administrative privileges undergo background checks before onboarding. Sensitive roles (security team, SRE, database administrators) receive enhanced screening. Background checks are refreshed every two years for personnel in sensitive positions. HR manages the process with results documented in the HRIS system, and the CISO defines which roles qualify as sensitive.
