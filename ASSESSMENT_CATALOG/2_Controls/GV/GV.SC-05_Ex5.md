# GV.SC-05 Ex5: Define SLAs for Supplier Monitoring

**Subcategory:** GV.SC-05 — Requirements to address cybersecurity risks in supply chains are established, prioritized, and integrated into contracts and other agreements with suppliers and other relevant third parties
**NIST SP 800-53 Ref:** SA-04, SA-09, SR-03, SR-05, SR-06, SR-10

## Implementation Example

> Define security requirements in service-level agreements (SLAs) for monitoring suppliers for acceptable security practices throughout the relationship.

## Alma Security Implementation

Alma Security includes security SLAs in vendor agreements: uptime requirements with security context (99.9% for Tier 1), vulnerability remediation timelines (critical: 48 hours, high: 7 days), incident notification windows, and compliance reporting cadences. Monitoring is continuous for Tier 1 vendors (automated security rating feeds) and quarterly for Tier 2 vendors. SLA compliance is tracked in ServiceNow with automated alerting for breaches.
