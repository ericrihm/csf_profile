# RS.MI-01 Ex3: Third-Party Containment Actions

**Subcategory:** RS.MI-01 — Incidents are contained

**NIST SP 800-53 Ref:** IR-04

## Implementation Example

> Allow a third party (e.g., internet service provider, managed security service provider) to perform containment actions on the organization's behalf

## Alma Security Implementation

Alma Security's incident response retainer with an external DFIR provider includes pre-authorized containment capabilities for incidents exceeding internal capacity. The retainer agreement defines escalation thresholds and containment authorities granted to the third party. Additionally, AWS support has pre-authorized access to assist with containment of infrastructure-level incidents such as compromised root accounts or large-scale DDoS attacks, coordinated through the AWS Enterprise Support channel.
