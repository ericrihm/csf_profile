# RS.CO-03 Ex2: Voluntary TTP Sharing with Sensitive Data Removed

**Subcategory:** RS.CO-03 — Information is shared with designated internal and external stakeholders

**NIST SP 800-53 Ref:** IR-04, IR-06, IR-07, SR-03, SR-08

## Implementation Example

> Voluntarily share information about an attacker's observed TTPs, with all sensitive data removed, to help others identify and respond to incidents more quickly

## Alma Security Implementation

Alma Security shares sanitized threat intelligence including attacker TTPs and IOCs with industry ISACs and trusted sector peers when incidents reveal novel attack techniques. All customer data, internal IP addresses, and organization-identifying information are stripped before sharing. The D&R team under Nadia Khan reviews outbound threat intelligence for sensitive data before publication. This practice supports the broader security community while building trust with industry partners.
