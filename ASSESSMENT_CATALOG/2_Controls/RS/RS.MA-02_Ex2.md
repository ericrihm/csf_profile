# RS.MA-02 Ex2: Incident Severity Estimation

**Subcategory:** RS.MA-02 — Incident reports are triaged and validated

**NIST SP 800-53 Ref:** IR-04, IR-05, IR-06

## Implementation Example

> Apply criteria to estimate the severity of an incident

## Alma Security Implementation

The incident response playbook defines severity estimation criteria based on data classification of affected systems, number of impacted users, service availability impact, and regulatory notification implications. SOC analysts apply the severity matrix during initial triage to assign severity levels (Critical, High, Medium, Low) that drive response urgency, resource allocation, and escalation requirements. Severity estimates are refined as investigation reveals additional scope information.
