# RS.AN-03 Ex3: Systemic Root Cause Analysis

**Subcategory:** RS.AN-03 — Analysis is performed to establish what has taken place during an incident and the root cause of the incident

**NIST SP 800-53 Ref:** AU-07, IR-04

## Implementation Example

> Analyze the incident to find the underlying, systemic root causes

## Alma Security Implementation

Post-incident reviews conducted by the D&R team identify systemic root causes beyond immediate technical failures, examining process gaps, control deficiencies, and organizational factors. Findings are documented in ServiceNow and tracked through remediation tickets with assigned owners and target dates. The Incident Response Enhancement project ($150K) was initiated specifically to address systemic gaps identified through root cause analysis of the 2024 security incidents.
