# DE.AE-03 Ex1: Centralized Log Aggregation

**Subcategory:** DE.AE-03 — Information is correlated from multiple sources

**NIST SP 800-53 Ref:** AU-06, CA-07, IR-04, IR-05, IR-08, PM-16, SI-04

## Implementation Example

> Constantly transfer log data generated by other sources to a relatively small number of log servers to simplify correlation

## Alma Security Implementation

Alma Security forwards CloudTrail logs, VPC Flow Logs, and DNS query logs to a centralized S3 bucket with CloudWatch Logs integration for unified querying. SentinelOne endpoint telemetry is aggregated in the SentinelOne management console, and O365 ATP logs are retained in the Microsoft 365 unified audit log. Full centralization into a single SIEM is not yet implemented; the planned custom cross-source correlation scripts will bridge the gap between AWS-native, endpoint, and email log repositories.
